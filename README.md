# 🎧 Rhizome Network 音乐社区

> 一款基于 **Vue 3 + Electron + Spring Boot** 开发的现代化桌面音乐播放器
> 支持本地音乐管理、全局持久播放、用户登录与云端数据同步

---

## 📖 项目简介

**Rhizome Network** 是一款采用**前后端分离架构**的桌面音乐应用，主打简洁统一的线框风格设计。

应用集成了本地音乐播放与云端数据同步能力，支持页面切换不中断播放、自动解析音频元数据、多主题切换，既适合作为日常听歌工具，也适用于毕业设计项目。

---

## ✨ 核心功能

### 🎵 本地音乐管理

* 支持批量导入音频文件
* 自动解析歌曲名、艺术家、时长等元数据
* 本地音乐列表持久化缓存

### ▶️ 全局持久播放

* 音频实例全局唯一
* 页面切换不中断播放
* 避免多音频实例冲突

### 👤 用户账号系统

* 登录状态统一管理
* 添加音乐时自动同步至后端数据库
* 支持云端数据持久化

### 🎨 多主题支持

* 暗黑 / 浅色双主题
* 高对比度线框 UI 风格
* 全局样式统一

### 🎚️ 播放控制

* 播放 / 暂停
* 重复点击切歌
* 列表自动去重
* 本地缓存不丢失

---

## 🛠️ 技术栈

### 前端

* **框架**：Vue 3 + Vite
* **状态管理**：Pinia
* **UI 组件库**：Element Plus
* **桌面端**：Electron
* **音频解析**：jsmediatags
* **样式方案**：CSS 变量 + 黑白线框主题

### 后端

* **框架**：Spring Boot
* **数据库**：MySQL
* **接口风格**：RESTful API

### 其他技术

* Electron IPC（主进程 / 渲染进程通信）
* localStorage 本地缓存
* 文件选择与 Blob 音频播放

## 📂 项目结构

```bash
rhizome-network/
├── src/
│   ├── views/            # 页面组件（本地音乐、个人中心、歌单等）
│   ├── stores/           # Pinia 全局状态（播放器、用户信息）
│   ├── composables/      # 全局组合式函数（主题、工具方法）
│   ├── api/              # 后端接口请求
│   └── App.vue           # 全局音频挂载入口
├── electron/             # Electron 主进程、preload 预加载脚本
├── backend/              # Spring Boot 后端服务
└── package.json          # 依赖与启动脚本
```

## 🚀 快速启动

### 📌 环境要求

* Node.js >= 16
* Java 21
* MySQL >=5.7

---

### 💻 前端启动

```bash
# 安装依赖
npm install

# Web 开发模式
npm run dev

# Electron 桌面端开发模式
npm run electron:dev

# 构建 Web
npm run build

# 打包桌面应用
npm run electron:build
```

---

### 🗄️ 后端启动

1. 创建 MySQL 数据库
2. 执行 SQL 初始化表结构
3. 配置 `application.yml` 数据库连接信息
4. 启动 Spring Boot 项目

## 🚀 项目亮点

1. **全局音频不中断**
   播放器脱离页面生命周期，切页持续播放

2. **音频元数据自动解析**
   自动识别歌曲名、歌手，避免“未知艺术家”

3. **无多音轨冲突**
   播放新歌曲时自动停止上一首

4. **极简线框 UI**
   高对比、无圆角、风格统一
